the ecosystem

One identity, three engines

The trick that keeps "many tools on many boards" from becoming twenty forks: a shared look + a shared tool suite, with the right base OS chosen per role.

routers

SpookyWrt

OpenWrt image-builder flow. Router / AP / travel / NAS across H68K, GL.iNet, NanoPi, Pi. LuCI underneath, GL.iNet-class UI on top.

diagnostics

SpookyProbe

Plug-in "network doctor in a box" for field techs. Debian + a containerized tool suite. Auto-sweep, pcap to NVMe, hand-off report.

security · authorized

SpookyLab

Home-lab / CTF / authorized-audit toolkit. Monitor-mode Wi-Fi, IDS, isolated lab VLAN, consent-gated. Your gear, your rules.

supported hardware

Pick your box

architecture

Three decoupled layers
